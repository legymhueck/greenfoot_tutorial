= Steckbrief der Methode *showText*

== Lernziel

Status- und Spielfeedback auf der Welt anzeigen (`showText`), zum Beispiel Koordinaten oder Punkte.

== Aus welcher Klasse stammt die Methode?

`World`

== Was macht die Methode?

* Zeigt Text auf der Welt an.
* Der x- und y-Wert auf der Welt repräsentiert die Mitte des Textes und
NICHT seine (linke) Anfangsposition!

== Wie sieht der Bauplan der Methode aus?

[source,java]
----
public void showText(String text, int x, int y)
----

== Was bedeuten die einzelnen Wörter?

[cols="1,4",options="header"]
|===
|Begriff |Bedeutung

|`public`
|Andere Klassen können auf die Methode zugreifen.

|`void`
|stammt die Methode. Sie gibt nichts zurück.

|`showText`
|Name der Methode.

|`String text`
|1. Parameter: Der anzuzeigende Text.

|`int x`
|2. Parameter: x-Koordinate der Textmitte.

|`int y`
|3. Parameter: y-Koordinate der Textmitte.
|===

Im folgenden Code werden die Koordinaten des Raumschiffs auf der Welt
angezeigt, wenn run() aufgerufen wird.


[source,java,linenumbers]
----
import greenfoot.*;

public class Weltraum extends World {
  private Raumschiff r;

  public Weltraum() {
    super(600, 400, 1);

    r = new Raumschiff();
    addObject(r, 100, 100);
  }

  // Die Koordinaten des Raumschiffes werden auf der Welt angezeigt.
  public void act() {
    showText("x:" + r.getX() + ", y:" + r.getY(), 100, 20);
  }
}

----

== Übungsaufgaben

1. Gib statt Koordinaten einen eigenen Status-Text aus.
2. Zeige zusätzlich die Rotation des Raumschiffes rechts neben seinen Koordinaten.
3. Zeige die Position eines zweiten Objektes an einer anderen Stelle.
4. Lösche `Private Raumschiff r` und erzeuge stattdessen das Raumschiff im Konstruktor.
Begründe, warum das nicht funktioniert.
